Команды в Shell

clear – очищает всю консоль
echo <text> – печатает текст в консоль
id – полное айди юзера
free -h – информация об RAM
df -h – информация о диске
uname -a – информация об устройстве (Время релиза, система, хост и т.д)
ps -ef – запущенные процессы
top – информация об процессах (она изменяется)
env – подробная информация об окружении экзекутора
ulimit -a – показывает лимиты памяти
python -m pydoc <moduleName> – тоже самое что и help
pip install <moduleName> – скачивает модуль для Python по его имени
pip list – получает список все скаченных модулей для Python
pip show <module_name> – получает подробную информацию о модуле (имя версия и т.д)
pip uninstall <moduleName> – удаляет модуль по его имени
lscpu – получает информацию об CPU
cat <file_path> – получает исходный код файла. Нужно разрешение приложения на файлы, чтобы можно было прочитать любой файл
cat /proc/cpuinfo – получает информацию о всех процессорах устройств
uname -a – информация об модели телефона
uptime – сколько работает данное устройство
ip – получает айпи адрес клиента. Требуется разрешение
pwd – получает текущий каталог
ls <path> – получает все файлы и папки в данном пути
rm <path> – удаляет файл
head <path> – получает первые 10 строк файла
tail <path> – получает последние 10 строк файла
less <path> – открывает файл для чтения с возможностью скроллинга
hostname – получает хост имя устройства
whoami – имя пользователя (типо u0_a123)
date – текущее время
touch <name> – создаёт новый файл в терминале с заданным именем
ping <hostname> – получает пинг хоста
netstat – получает информацию о сети
curl <url> – делает запрос на данный URL
history – получает все команды которые ты использовал в данной сессии. Чтобы очистить историю используй history -c
time <command> – запускает данную команду и потом показывает за сколько времени она была завершена
vmstat – статистика виртуальной памяти. Требует разрешения
lsof – список доступных файлов
type <command> – получает тип команды. Может вывести <command> is a shell built-in
which <command> – получает путь к этой команде
whereis <command> – получает все пути к этой команде
help – получает все внутренние команды
sleep <second> – останавливает поток на установленные секунды
bc – включает режим калькулятора. Если ввести слишком большие числа, то у тебя зависнет команда или вообще весь терминал. Чтобы отключить его введи quit
<command> --version – получает информацию о данной команде
<command> --help – получает документацию об команде с доступными опциями
du <path> – получает размер каталога
python – позволяет использовать Python. Чтобы прекратить это нужно использовать quit
pkg list-all – получает все команды, которые можно скачать
pkg install <command> – скачивает данную команду для bash
pkg list-installed – получает все скачанные команды
set – получает все глобальные переменные в данной сессии
unset <global_var> – убирает глобальную переменную
local – создаёт локальную переменную. Её можно использовать только в функциях
unzip <path> – декомпилирует ЗИП файл и отображает его содержимое
stat <option> <mode> <path> – получает статистику файла. Например (stat -c %s <path>) получает размер файла в байтах 
chmod <perm> <path> – устанавливает разрешения файлу. Есть три вида: создатель (файла), группа (вставленные юзеры) и остальные. Например если в rules поставить 770 то это означает что автор и группы могут делать с файлом что угодно, а остальные ничего не могут
compgen -c – получает список всех доступных команд
cal – мини календарь
exit <number> – выходит из терминала с заданным returncode (по умолчанию 0, что значит что сессия успешная)
logout – просто закрывает сессию
tr – меняет символы на другие символы. Например (echo "hello" | tr 'a-z' 'A-Z') меняет все строчные на заглавные буквы
nice – команда которая задаёт приоритет скрипту для CPU. Например (nice -n -20 python script.py)
tty – показывает реальный путь к твоей сессии
ul – это очень старая функция с 70-80х годов и про неё нету информации. Она превращает символы нижнего подчёркивания в понятные для терминала, но теперь это уже не нужно. Пример: (echo "test_word" | ul)
setarch – заставляет думать программу, что она запускается на другой машине. Например (setarch i386 ./my_32bit_program или setarch i386 uname -m)
nc – сетевая утилита, которая читает  и пишет данные через TCP или UDP соединения
am – очень мощная функция. Позволяет управлять тебе приложениями. Например (am start -n com.android.settings/.Settings) заставляет тебя зайти в настройки твоего телефона
logcat – логирует все события происходящие на твоём устройстве. Чтобы очистить активируй logcat -c. Можно логировать события определённого файла с помощью (logcat | grep file.txt)
script – записывает всё что ты делаешь в терминале в файл "typescript"
tload – рисует график нагрузки. Если он зашкаливает, то значит, что терминал жрёт много памяти
editor – считай блокнот
logname – тоже самое что и whoami
tset – команда контроля над терминалом
yes – бесконечно спамит заданным символом
watch <command> – показывает все логи команды
getprop – специальная android информация которая выдает очень много информации об устройстве
shred – удаляет файл (как rm), но делая так чтобы его было нереально восстановить 
lli – это JIT компилятор для LLVM с помощью которого можно запускать скрипты практически не компилируя его

cp Define.s /sdcard/Download – копирует файл и переносит его в папку Download
find /sdcard/Download -type f – ищет только файлы в папке
find /sdcard/Download -name "*.txt" – идёт все файлы у которых есть .txt
cat /dev/urandom | tr -dc 'a-z0-9' | head -c 10 – создаёт рандомную строку с размером 10 символов
curl -s http://example.com/script.sh | bash – загружает скрипт из ссылки и запускает его в терминал
echo "Uuieks" > TestFile.txt – изменяет исходный код TestFile.txt. Если использовать >> то к файлу добавиться этот текст, а не полностью замениться
am start -n com.android.settings/.Settings – заставляет тебя зайти в настройки
