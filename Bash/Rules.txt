Bash

1) Указание переменной всегда должна быть слитной (myvar="1234")
2) Чтобы получить значение переменной нужно использовать знак доллара $ (например $EUID, ls $(pwd))
3) В функции ты можешь использовать $1, $2, $3 ... чтобы получить конкретный аргумент. Ты можешь использовать $@ чтобы получить все параметры, а $# чтобы получить количество всех параметров. В функциях нельзя делать аргументы функции, как это обычно делают 
4) По умолчанию все переменные глобальные и чтобы сделать переменную в конкретном окружении нужно использовать local (local myvar="12345")
5) Первая строка любого скрипта (файла .sh) всегда должна начинаться на #!/bin/bash
6) Терминал использует цвета в виде ANSI формата
7) Можно получить код ассемблера в Shell, но чтобы его посмотреть тебе сначала нужно получить сам этот скрипт интерпретатора (например Python)
8) Bash (Shell) файлы обозначаются как .sh
9) Операторы: ! [[  ]] – не; || – или; ; – последовательность
10) Stdout (Standard Output) – то, что печатает команда. Следовательно stderr это ошибка
11) Для файлов существует понятие "владелец". Можно получить собственный юзер с помощью id -u. Можно указывать права файла с помощью chmod. Типо chmod 600 TestFile.txt делает так, чтобы тот, кто создал файл мог читать и редактировать файл, а группы (разрешённые юзеры) и остальные ничего не могли с ним делать
12) Ты можешь получить все глобальные переменные с помощью команды set и убрать из с помощью unset (если создавать в функции то используй export var="10")
13) Опции в командах можно накладывать (типо использовать id -an)
